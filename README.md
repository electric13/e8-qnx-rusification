# KZB Tools for Geely Galaxy E8 (QNX Resources)

Доработанный набор инструментов для распаковки и запаковки ресурсов формата **KZBF**, используемых в автомобилях Geely Galaxy E8 (платформа QNX).

Данный проект основан на [Python-KZB](https://github.com/jakka351/Python-KZB), но существенно модифицирован для корректной работы с современными архивами Geely:
- Исправлена поддержка формата **KZBF** (плоская структура).
- Добавлена автоматическая обработка проприетарных заголовков изображений (24-байтовый wrapper для `.png`).
- Добавлена система метаданных (`_kzbf_meta.json`) для точного восстановления оригинальной структуры архива при запаковке.
- Реализован автоматический пересчет смещений (offsets) при изменении длины строк локализации.

---

## ⚠️ Внимание!
Все операции по замене системных файлов производятся на ваш страх и риск. Ошибка при работе с консолью QNX может привести к невозможности загрузки мультимедийной системы. Всегда делайте резервные копии!

---

## Порядок работы (Русификация)

### 1. Подготовка
Извлеките файл ресурсов из системы. В Geely Galaxy E8 (официальные дилеры в РБ) основной файл — `e171.kzb`.
- **Важно:** Имя файла актуально для машин от официальных дилеров в РБ. Для машин из Китая имя может отличаться (не проверялось из-за отсутствия доступа к таким авто).
- В образе прошивки: `/bin/kanzi/e171.kzb`
- В работающей машине: `/mnt/bin/kanzi/e171.kzb`

### 2. Распаковка
Запустите `kzb.py` и используйте вкладку **Unpack**. 
Скрипт создаст директорию `e171.kzb_unpacked`, в которой:
- Находятся все извлеченные ресурсы.
- Создан файл `_kzbf_meta.json` (важен для последующей сборки, не удаляйте его).

### 3. Редактирование локализации
Файлы локализации обычно имеют вид `r__Localization_...` без расширения.
- Например, `r__Localization_DriveInfo_Local_English` отвечает за данные в правой части водительского экрана.
- **Правила редактирования:**
  - Строки разделяются нулевым байтом (`\x00`).
  - Можно использовать **русский язык** (кодировка UTF-8).
  - Длина строк может меняться — упаковщик сам пересчитает размеры.
  - Рекомендуется использовать редакторы, отображающие служебные символы (Midnight Commander, VS Code, Hex-редакторы).

### 4. Работа со шрифтами
Шрифты хранятся с 8-байтовым заголовком Kanzi, что мешает их просмотру в системе.
Чтобы проверить наличие кириллицы в шрифте, сделайте "чистую" копию для просмотра:
```bash
dd if="path/to/font.ttf" of="clean_font.ttf" bs=1 skip=8
```
*Примечание:* 
- Не заменяйте оригинальный файл в папке распаковки чистой копией, иначе архив может не собраться.
- **Обязательно проверяйте шрифт:** В некоторых шрифтах набор русских букв сильно обрезан, и вывести ими произвольную надпись нельзя. Сравнивайте визуально с фотографиями исходного экрана, чтобы убедиться в наличии нужных символов.

### 5. Запаковка
Используйте вкладку **Pack** в `kzb.py`, указав папку `e171.kzb_unpacked`.
Скрипт создаст новый `.kzb` файл, соблюдая оригинальное выравнивание (4096 байт) и структуру KZBF.

## Установка в автомобиль

1. **Загрузка через Android:**
   Подключитесь по ADB и отправьте файл в общую папку:
   ```bash
   adb push e171.kzb /mnt/vendor/var_log/e171.kzb
   ```
   *(Папка `/mnt/vendor/var_log/` в Android соответствует `/var/log/` в QNX).*

2. **Работа в консоли QNX (UART):**
   Разрешите запись в системный раздел:
   ```bash
   mount -u -o /mnt
   ```
   Сделайте бэкап оригинала:
   ```bash
   mv /mnt/bin/kanzi/e171.kzb /mnt/bin/kanzi/e171.kzb_old
   ```
   Установите новый файл:
   ```bash
   mv /var/log/e171.kzb /mnt/bin/kanzi/
   sync
   ```
3. **Перезагрузка:**
   Выполните перезагрузку системы. Если все сделано правильно, отобразятся новые надписи.

---

## Откат (если что-то пошло не так)
Если система не загрузилась или интерфейс работает некорректно, подключитесь через UART и выполните:
```bash
mount -u -o /mnt
rm /mnt/bin/kanzi/e171.kzb
cp /mnt/bin/kanzi/e171.kzb_old /mnt/bin/kanzi/e171.kzb
sync
```
После чего перезагрузитесь.

## 💡 Важные замечания
Эта инструкция написана вручную. Команды не копировались из реальной консоли "в лоб", поэтому возможны опечатки. 
- Обязательно проверяйте, что вы понимаете смысл каждой команды перед её выполнением. 
- Ошибка может превратить систему автомобиля в "тыкву". Будьте предельно аккуратны!
